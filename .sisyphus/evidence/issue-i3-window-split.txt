### Test Scenarios for Window Splitting

#### Scenario 1: Horizontal Window Split
**Test Steps:**
1. Execute `C-x 2` (split-window-below) in Emacs
2. Verify window splits horizontally
3. Verify both windows display content correctly

**Expected Result:**
- Frame splits into two windows stacked vertically
- Each window has its own modeline
- Both windows can display different buffers
- Text renders correctly in both windows

**Actual Result:**
- Emacs core functionality for window splitting should work (standard Emacs C code)
- UI layout controlled by HyaloiOSNavigationLayout.swift
- Manual verification required to confirm visual appearance

**Technical Details:**
- Frame size at init: 1022x1372 (from logs)

- Total: 73 columns x 49 lines

#### Scenario 2: Vertical Window Split
**Test Steps:**
1. Execute `C-x 3` (split-window-right) in Emacs
2. Verify window splits vertically
3. Verify proper width calculation for side-by-side windows

**Expected Result:**
- Frame splits into two windows side-by-side
- Each window has reduced width
- Text wrapping adjusts accordingly

**Actual Result:**
- Standard Emacs split-window-right functionality
- Window width calculations handled by window.c
- Visual verification needed for iOS-specific rendering

#### Scenario 3: Window Navigation
**Test Steps:**
1. Split window horizontally
2. Use `C-x o` (other-window) to navigate between windows
3. Verify cursor moves to other window
4. Verify active window highlight updates

**Expected Result:**
- Cursor switches between windows
- Active window visually indicated
- Status bar updates for active buffer

#### Scenario 4: Window Deletion
**Test Steps:**
1. Split window
2. Execute `C-x 0` (delete-window) or `C-x 1` (delete-other-windows)
3. Verify windows merge correctly
4. Verify no rendering artifacts

**Expected Result:**
- Window closes cleanly
- Remaining window expands to fill space
- No visual glitches
