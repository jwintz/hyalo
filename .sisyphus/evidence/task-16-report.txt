ISSUE ANALYSIS
--------------
The app terminates after Emacs initialization completes. This appears to be
expected behavior based on the current iOS implementation:

1. HyaloApp.swift uses HyaloRootView() from HyaloKitFramework
2. The framework initializes Emacs in a background thread
3. Emacs initializes successfully but the UI framework is not fully implemented
4. The app exits because there's no running event loop keeping it alive

This is confirmed by examining the iOS source files:
- iOS/HyaloApp/Sources/HyaloApp.swift: Thin wrapper around HyaloKitFramework
- iOS/HyaloApp/HyaloApp.swift: Contains placeholder ContentView

The iOS implementation is in early stages - the app launches, Emacs initializes,
but the UI layer (SwiftUI views, command palette, file browser) is not yet
connected to the running Emacs instance.
