### Technical Notes

#### Channel Bridge Architecture
The channel bridge enables bidirectional communication:
- **Swift -> Emacs**: Via `ChannelBridge.enqueueCommand()` which calls `hyalo_ios_pop_command` 
- **Emacs -> Swift**: Via `hyalo-ios-dispatch` DEFUN in iosfns.m

#### Navigator Buffer List Flow
From the architecture documentation:
```
User taps buffer in Navigator -> Channel callback -> switch-to-buffer -> Emacs -> window-buffer-change-hook -> hyalo-navigator-update-buffers -> Swift UI update
```

#### Known Limitations
- Reverse channel (Swift -> Emacs) queue has bounds that need verification
- Navigator selection has race condition guards in place
- Stale callback filtering implemented in ProjectNavigatorViewModel
